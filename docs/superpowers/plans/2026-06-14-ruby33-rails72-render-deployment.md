# Ruby 3.3 / Rails 7.2 Upgrade & Render Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade Epic Beer Run BE from Ruby 2.6.1/Rails 6.0.3 to Ruby 3.3.6/Rails 7.2, write real controller tests, and deploy to Render's free tier.

**Architecture:** Direct version jump — update Ruby and Rails in one pass, swap gem dependencies to compatible versions, write integration tests to confirm the endpoints work on the new stack, then add Render deployment files (Procfile, build script, render.yaml). The app is a 2-action API-only Rails app so upgrade risk is minimal.

**Tech Stack:** Ruby 3.3.6, Rails 7.2, PostgreSQL (pg ~> 1.5), Puma 6.4, Minitest (built-in), Render free tier, rbenv

> **All commands in this plan must be run inside WSL**, since the project lives at `~/VM_Home/development/Epic-Beer-Run-BE`. Open a WSL terminal and `cd ~/VM_Home/development/Epic-Beer-Run-BE` before starting.

- [ ] **Step 2: Write integration tests for the breweries endpoints**

Replace `test/controllers/breweries_controller_test.rb` with:

```ruby
require 'test_helper'

class BreweriesControllerTest < ActionDispatch::IntegrationTest
  test "GET /breweries returns 200 and an array" do
    get breweries_url
    assert_response :success
    body = JSON.parse(response.body)
    assert_kind_of Array, body
  end

  test "GET /breweries includes fixture breweries" do
    get breweries_url
    assert_response :success
    body = JSON.parse(response.body)
    names = body.map { |b| b["name"] }
    assert_includes names, breweries(:one).name
    assert_includes names, breweries(:two).name
  end

  test "GET /breweries/:id returns 200 and the correct brewery" do
    brewery = breweries(:one)
    get brewery_url(brewery)
    assert_response :success
    body = JSON.parse(response.body)
    assert_equal brewery.id, body["id"]
    assert_equal brewery.name, body["name"]
    assert_equal brewery.state, body["state"]
  end

  test "GET /breweries/:id with unknown id returns 404" do
    get brewery_url(id: 0)
    assert_response :not_found
  end
end
```

## Task 6: Add Render deployment files

**Files:**
- Create: `Procfile`
- Create: `bin/render-build.sh`
- Create: `render.yaml`

- [ ] **Step 1: Create `Procfile`**

```
web: bundle exec puma -C config/puma.rb
```

- [ ] **Step 2: Create `bin/render-build.sh`**

```bash
#!/usr/bin/env bash
set -o errexit
bundle install
bundle exec rails db:migrate
```

- [ ] **Step 3: Make `bin/render-build.sh` executable**

```bash
chmod +x bin/render-build.sh
```

- [ ] **Step 4: Create `render.yaml`**

```yaml
services:
  - type: web
    name: epic-beer-run-be
    env: ruby
    buildCommand: "./bin/render-build.sh"
    startCommand: "bundle exec puma -C config/puma.rb"
    envVars:
      - key: DATABASE_URL
        fromDatabase:
          name: epic-beer-run-db
          property: connectionString
      - key: RAILS_MASTER_KEY
        sync: false
      - key: RAILS_ENV
        value: production
      - key: RAILS_LOG_TO_STDOUT
        value: "true"

databases:
  - name: epic-beer-run-db
    databaseName: epic_beer_run
    user: epic_beer_run
    plan: free
```

> `RAILS_MASTER_KEY` is marked `sync: false` so Render prompts you to enter it manually — you'll paste the value from `config/master.key`.

- [ ] **Step 5: Commit**

```bash
git add Procfile bin/render-build.sh render.yaml
git commit -m "feat: add Render deployment files (Procfile, build script, render.yaml)"
```

## Task 7: Push and set up Render

- [ ] **Step 1: Push the branch to GitHub**

```bash
git push -u origin v2/dependency-upgrades
```

- [ ] **Step 2: Create a new Web Service on Render**

1. Go to [render.com](https://render.com) → New → Web Service
2. Connect your GitHub repo (`Epic-Beer-Run-BE`)
3. Select the `v2/dependency-upgrades` branch
4. Render will detect `render.yaml` and pre-fill the config

- [ ] **Step 3: Add the RAILS_MASTER_KEY environment variable**

In the Render dashboard for this service, go to **Environment** and add:
- Key: `RAILS_MASTER_KEY`
- Value: the contents of `config/master.key` (do NOT commit this file)

- [ ] **Step 4: Deploy**

Click **Deploy**. Render will:
1. Run `bin/render-build.sh` (bundle install + db:migrate)
2. Start Puma via `bundle exec puma -C config/puma.rb`

Watch the deploy logs — a successful deploy ends with:
```
==> Your service is live 🎉
```

- [ ] **Step 5: Smoke-test the live API**

```bash
curl https://<your-render-url>.onrender.com/breweries
```

Expected: JSON array of breweries (empty array `[]` is fine if the production DB is empty — the schema will be correct).
